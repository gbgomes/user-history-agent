Descrição:
Este processo permite a importação de um arquivo texto contendo o cadastro de clientes. Durante o processo de importação o cliente pode ser identificado de duas formas: Código ou CPF/CNPJ (Código CVM). Caso a opção de identificação seja CPF/CNPJ( Código CVM) e a flag "Importar INR sem CPF/CNPJ informado" esteja marcada, clientes que possuirem CPF/CNPJ inválido e o perfil CVM seja "investidor não residente", a importação irá cadastrar o código CVM no lugar do CPF/CNPJ, cujo o valor virá no mesmo campo de importação.

O layout do arquivo a ser importado é definido no Cadastro de Processos de Importação de Cadastro de Clientes.

Os campos não indicados possuem tamanho fixo de 1 caracter.
Os campos que contêm caracteres quando ultrapassarem o tamanho máximo serão truncados à direita.
Quando o tamanho for inferior, serão acrescidos espaços, também a direita.
Os campos que contém dígitos quando ultrapassarem o tamanho máximo serão zerados, podendo, portanto, invalidar o registro do cliente que estiver sendo importado, ficando registrado, se for o caso, no LOG do processo.
O número máximo de dígitos na máscara do Código do Cliente é 15.
Se no Processos de Importação (PI) o flag "Descarta Header e Footer" estiver marcado, a primeira e última linha do arquivo serão descartadas.

Inicialmente o Sistema solicita a seleção de um dos PI's previamente cadastrados.
Solicita, também, autorização para sobrescrever o cadastro a partir das informações do arquivo texto a ser importado. Em caso afirmativo, o Sistema atualizará as informações dos clientes que já existam no Cadastro e incluirá os clientes novos. Caso Contrário, o Sistema só realizará a última tarefa.
Segue-se a importação propriamente dita do arquivo. Serão importados somente os registros válidos, ou seja, os registros cujos campos obrigatórios estejam preenchidos. Caso haja algum problema ou aviso, estes serão coletados em um arquivo LOG de importação que poderá ser consultado ou impresso.

O processo de conversão dos diversos campos é efetuado como segue:

  • Nos campos CEP, CPF/CGC e Banco, não são considerados caracteres não numéricos tais como espaços, vírgula e pontos.
  • Nos campos de flag são considerados apenas os seguintes caracteres: 'S', 'Y' e '1', em caso afirmativo, e qualquer outro caracter em caso negativo.
  • Quando os campos Literal PF e Literal PJ não estiverem configurados, o campo de Pessoa Jurídica deverá conter o caracter ‘1’ para Pessoa Física e ‘2’ para Pessoa Jurídica, caso contrário deverá estar preenchido conforme o caracter informado em um dos campos.
  • Quando a opção "Identifica cliente por CPF/CNPJ" estiver selecionada, o código do cliente será gerado pelo sistema.
  • O novo parâmetro Status não é obrigatório. Caso não esteja configurado no Cadastro de Processo de Importação de Cadastro de Clientes, os clientes são importados com o status Ativo.
  • Se o campo "Posição" correspondente ao Status estiver configurado no Cadastro de Processo de Importação de Cadastro de Clientes, a importação exige um dos literais cadastrados (Literal Ativo, Literal Inativo e Literal Encerrado). Caso identifique caracteres diferentes dos literais cadastrados, é apresentada uma mensagem de erro e a importação do cadastro do cliente em questão é abortada.

Caso o parâmetro "Arquivo CBLC" esteja marcado no Cadastro de Processo de Importação de Cadastro de Clientes, a importação realizará os seguintes procedimentos:

  • Os clientes importados serão associados ao Assessor Bolsa e ao Assessor 2 Bolsa definidos no Processo de Importação. Portanto, os clientes informados no arquivo de importação não necessitarão da informação do Assessor e do Assessor 2.
  • Na importação de inclusão (com o parâmetro Sobrescreve no Cadastro desmarcado), caso já exista um cliente cadastrado com o CPF/CNPJ igual ao informado no arquivo e que não seja um Cliente Bovespa, um novo cliente será incluído com o mesmo CPF/CNPJ, cadastrando-o como um "Cliente Bovespa".
  • Caso o cliente já cadastrado seja um Cliente Bovespa, será emitida uma mensagem no LOG de importação e o cliente não será importado.
  • Se não houver nenhum cliente cadastrado com o CPF/CNPJ informado no arquivo, o sistema cadastrará um novo cliente com o campo "Cliente Bovespa" igual a "Sim".
  • A importação de alteração (com o parâmetro Sobrescreve no Cadastro marcado), para o cliente com CPF/CNPJ igual ao informado no arquivo, irá alterar somente o cliente com este CPF/CNPJ e com o campo "Cliente Bovespa" igual “Sim”.

Os arquivos da CBLC são identificados pelo Header, onde, na posição 11 encontra-se o nome "CBLC".

    Observações:

    1. Caso a Conferência de Pendência esteja ativada, os dados cadastrais dos clientes informados na importação ficarão pendentes de conferência. Desta forma, somente será possível atualizar qualquer informação cadastral destes clientes após realizar a conferência.
    2. Os campos "Conferência de Pendência por lote" e "Justificativa para Aprovação por lote", só serão exibidos caso a Conferência de Pendência esteja ativada nos parâmetros do sistema.
    3. Caso o cliente informado no arquivo esteja pendente de conferência, somente será possível visualizar ou atualizar qualquer informação cadastral deste após realização da conferência.

    *Atenção: Informação apenas cadastral, aguardando liberação de suporte no processamento das Amortizações, Resgates e Distribuição de Resultados.
